# 第3章 基本機能

> 【tier: 無料】友だち管理・配信・自動応答・リッチメニューの 4 機能を、ボタン単位で「押すと何が起きるか」を理解する。

## 章の目的

- 友だち一覧・タグ・セグメントの基本オペレーションを迷わず実行できる
- 配信（一斉 / セグメント / トリガー）の違いと管理画面上の操作手順を理解する
- 自動応答（キーワード応答・ステップ配信）の最小設定を自分でできる
- リッチメニュー画像の差し替え・タップ領域設定を一巡できる

## 想定読者

- セットアップが終わり、まず「触ってみたい」運用担当者
- 既存 SaaS（Lstep / Utage）から移行してきて、UI のマッピングを掴みたい人

## 目次

- 3.1 友だち管理（タグ・セグメント・UUID）
- 3.2 配信（一斉 / セグメント / トリガー / 予約）
- 3.3 自動応答（キーワード応答・ステップ配信）
- 3.4 リッチメニュー（画像差し替え・領域・出し分け）
- 3.5 ダッシュボードの数字の読み方（基礎）

## 前提

- 第 2 章でセットアップが完了している
- 自社 LINE に最低 1 人の友だちがいる（テスト用に自分自身を追加でも可）

## 次の章

第 4 章（🔒 有料）では、ここまでの基本機能を「獲得につながる設計」に組み替えるためのキャンペーン設計論に進みます。トラッキングリンク必須の理由と、診断 → セグメントへの落とし方を扱います。
