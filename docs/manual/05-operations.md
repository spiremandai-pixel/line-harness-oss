# 第5章 運用フロー

> 【tier: 🔒 有料】週次 A/B テスト・BAN 対策・配信ガバナンスを含む、事故らない継続運用のレール。

## 章の目的

- 週次サイクルで A/B テストを回し、配信文・件名・CTA を継続的に改善する型を持てる
- LINE 公式アカウントの BAN / 制限リスクを回避する配信頻度・文面・URL の運用ルールを敷ける
- 複数人運用時の「誰が何を配信していいか」の権限・確認フローを設計できる
- 配信前チェックリスト・配信後レビューフォーマットを自社に導入できる

## 想定読者

- キャンペーンを 1 本回し終えて、継続運用フェーズに入った事業者
- 複数のクライアント LINE を JV / 代行で運用している事業者
- 「送信事故」「BAN リスク」「権限事故」を仕組みで潰したい運用責任者

## 目次

- 5.1 週次 A/B テストのサイクル設計（仮説 → 配信 → 集計 → 反映）
- 5.2 BAN / 制限対策（頻度・文面・URL・苦情率）
- 5.3 権限・確認フロー（誰が押せるか / 二重確認 / 送信前プレビュー）
- 5.4 配信前チェックリスト・配信後レビュー
- 5.5 トラブル時のロールバック手順

## 前提

- 第 4 章でキャンペーン設計を 1 本以上動かしている
- 第 7 章の「メッセージ送信は必ず事前確認」「.env.production 触るな」を読んでいる

## 次の章

第 6 章（🔒 有料）では、ここまでの運用フローを MCP / Claude Code に委譲し、半自動運用にスケールさせるパターンを扱います。
