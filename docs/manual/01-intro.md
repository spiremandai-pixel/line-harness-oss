# 第1章 イントロ

> 【tier: 無料】LINE Harness が解決する課題と、Lstep / Utage との立ち位置の違いを掴む。

## 章の目的

- LINE Harness が「何のための OSS か」を1文で説明できるようになる
- Lstep / Utage と LINE Harness の違いを「料金・ロックイン・AI 統合・UUID」の4軸で語れるようになる
- 本マニュアルの無料 / 有料の境界と読み進め方を理解する

## 想定読者

- これから LINE 公式アカウントを本気で運用したい個人事業主・中小事業者
- 既に Lstep / Utage を使っているが、料金・ロックイン・AI 連携の壁にぶつかっている運用担当
- 自社プロダクトで LINE 連携機能を作りたいエンジニア兼マーケター

## 本文

LINE 公式アカウントは、メールよりも開封され、SMS よりも文脈を残せる「最後に残った 1 対 1 のチャネル」です。一方で、配信・自動応答・リッチメニュー・タグ管理を本気で運用しようとすると、Lstep / Utage のような外部 SaaS に依存する以外の選択肢がほとんど存在しませんでした。SaaS は便利ですが、月額コストは規模に比例して跳ね上がり、データはベンダーの中に閉じ込められ、AI（Claude / GPT 等）を運用に組み込もうとした瞬間に「API がない」「自動化を許していない」という壁にぶつかります。

LINE Harness は、この閉塞を OSS で解く試みです。Cloudflare Workers + D1 上に最小コストで自前の運用基盤を立て、友だち管理・配信・自動応答・リッチメニュー・トラッキングまでを自社の手元で動かします。料金は Cloudflare の従量課金のみで、データは自分の D1 に残ります。

Lstep / Utage との違いは 4 軸で整理できます。

1. **料金** — 月額固定の SaaS 課金から、Cloudflare 従量課金 + 自社運用工数モデルへ
2. **ベンダーロックイン** — シナリオ・配信履歴・タグはすべて自分の D1。エクスポート可否を気にせず移行できる
3. **AI 統合性** — MCP 経由で Claude Code から直接「配信を作って投げる」「分析して次のシナリオを書く」ができる
4. **UUID 統合** — LINE / IG / X / LMS など複数チャネルのユーザーを 1 つの UUID で串刺しに管理できる前提で設計

つまり LINE Harness は、単なる Lstep クローンではなく **「AI が LINE を安全に操作するための基盤」** です。AI が暴走しないためのトラッキング・確認フロー・BAN 対策が運用層に組み込まれており、「自動化したいが事故も怖い」という現実的な要請に応えます。

最後に本マニュアルの境界を明示します。**第 1〜3 章・第 7 章は無料**で、「ボタンを押すと何が起きるか」を網羅します。**第 4〜6 章は有料（🔒）**で、「どう設計すれば獲得につながるか・どう運用すれば事故らないか・どう AI に任せるか」という How / Why を扱います。まずは無料章で自分の LINE を立ち上げ、数字が見えてから有料章に進む順序を推奨します。
