# 第6章 MCP × Claude Code で自動化

> 【tier: 🔒 有料】MCP 経由で Claude Code に LINE 運用を委譲する。配信案生成 → 確認 → 送信までを半自動化する設計パターン。

## 章の目的

- LINE Harness の MCP サーバーを Claude Code に接続し、ツール呼び出しから配信を組める
- 「AI が暴走しないための確認フロー」を MCP ツール設計レベルで担保できる
- 週次の集計・配信案生成・A/B テスト分析を Claude Code に委譲する型を持てる
- MCP ツールの権限境界（読み取り専用 / 書き込み / 送信）を明確に分離して設計できる

## 想定読者

- 第 5 章までで運用フローが回っている事業者
- 自社運用の工数を削減したい / クライアント数をスケールさせたい代行・JV 事業者
- Claude Code / MCP の基礎は触ったことがあるエンジニア兼マーケター

## 目次

- 6.1 MCP サーバーのセットアップと接続
- 6.2 ツール権限境界（read / write / send の 3 層分離）
- 6.3 配信案の自動生成 → 人間レビュー → 送信のフロー
- 6.4 週次インサイト集計を Claude Code に投げる定型プロンプト

- Claude Code（CC Max プラン）が手元で使える
